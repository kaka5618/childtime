## 一、卡池设计原则

「星梦泡泡」卡池围绕“泡泡精灵 + 星星触角 + 梦境材质”展开。所有角色都必须符合：

- 原创非人形泡泡精灵。
- 圆滚滚、软乎乎、儿童友好。
- 至少具备一个星星触角或星星内核。
- 不出现武器、战斗、恐怖、人脸、知名 IP 近似形象。
- 卡面不生成可读文字，名称和编号由前端或模板后期叠加。

稀有度分层：

- N：基础泡泡，情绪、颜色、云朵、糖果等简单概念。
- R：加入果冻、透明、星星内核、小天气、小道具。
- SR：月光、梦境、彩虹、星尘等更强主题。
- SSR：套系核心角色，具备专属传奇感和最高级边框。

## 七、数据库映射建议

每张卡可映射到 `cards` 集合：

```js
{
  card_id: 'bubble_n_001',
  series_id: 'star_dream_bubble',
  number: 1,
  name: '棉花泡泡',
  rarity: 'N',
  image_url: '',
  back_image_url: '',
  prompt_fragment: 'round fluffy cloud-like body, cream white and pale pink, tiny star antenna',
  sort_order: 1
}
```

生成正式 seed 数据时，建议保留 `prompt_fragment` 作为内部字段，便于后续重新生成或迭代卡图。

## 八、生成顺序建议

建议先按以下顺序生成和审查：

1. 每个稀有度先生成 2 张：No.01、No.21、No.37、No.47，以及 No.08、No.31、No.45、No.50。
2. 确认四档视觉梯度稳定后，再批量生成剩余卡。
3. 每批不超过 8 张，避免风格漂移后返工过大。
4. 每张图生成后按 `星梦泡泡美术规范.md` 的生产检查标准筛选。
