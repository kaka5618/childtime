# 云同步接入说明

当前小程序端已经有“微信登录”和“同步云端”入口，并已打开云同步开关。完成下面配置和部署后即可同步。

## 1. 开通云开发

在微信开发者工具中开通云开发环境，并记录环境 ID。

## 2. 配置项目云函数目录

项目已在 `project.config.json` 顶层配置：

```json
{
  "cloudfunctionRoot": "cloudfunctions/"
}
```

## 3. 部署云函数

右键下面两个云函数目录，分别选择“上传并部署：云端安装依赖”。

```text
cloudfunctions/syncUserData
cloudfunctions/getUserData
```

云函数会写入两个集合：

- `users`：孩子档案、当前主题、设置等轻量信息。
- `user_sync_snapshots`：任务、收藏、卡包状态的完整同步快照。

## 4. 云环境 ID

当前 `miniprogram/app.js` 的 `cloudEnvId` 留空，会使用开发者工具当前默认云环境。

如果你有多个云环境，建议填写指定环境 ID：

```js
cloudEnvId: '你的环境ID'
```

云同步开关当前为：

```js
cloudSyncEnabled: true
```

## 5. 验证

进入设置页，先点“微信登录”，再点“同步云端”。成功后设置页会显示最近同步时间。

如需验证恢复功能，可在设置页点“云端恢复”。它会读取 `user_sync_snapshots` 中当前微信用户最近一次同步快照，并覆盖本机任务、收藏、孩子档案和开包状态。

## 6. 上传高清卡图到云存储

小程序代码包内只保留压缩发布图。高清卡图建议上传到云存储，并按下面目录保存：

```text
card-assets/cards/star_dream_bubble/bubble_n_001.jpg
card-assets/cards/star_dream_bubble/bubble_n_002.jpg
...
card-assets/cards/star_dream_bubble/bubble_ssr_050.jpg
```

上传后，在 `miniprogram/app.js` 中配置云存储根 fileID：

```js
cloudAssetBaseFileID: 'cloud://你的环境ID.你的存储桶名/card-assets'
```

页面会自动拼出：

```text
cloud://你的环境ID.你的存储桶名/card-assets/cards/star_dream_bubble/bubble_n_001.jpg
```

如果个别文件路径不规则，可以用 `cloudAssetFileIDs` 单独指定：

```js
cloudAssetFileIDs: {
  bubble_ssr_047: 'cloud://你的环境ID.xxxx/custom/bubble_ssr_047.jpg'
}
```

没有配置云存储 fileID 时，页面会使用代码包内的极小占位图，避免质量扫描因为卡图资源过大失败。

当前项目已配置为：

```js
cloudAssetBaseFileID: 'cloud://cloudbase-d3g0zqjt752827234.636c-cloudbase-d3g0zqjt752827234-1435290004/card-assets'
```
